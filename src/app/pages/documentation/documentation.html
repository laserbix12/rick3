Documentación

Introducción

Esta documentación te ayudará a familiarizarte con los recursos de la Rick and Morty API y te mostrará cómo realizar diferentes consultas para que puedas sacarle el máximo provecho.

GraphQL

https://rickandmortyapi.com/graphql

GraphQL 
query {
  characters(page: 2, filter: { name: "rick" }) {
    info {
      count
    }
    results {
      name
    }
  }
  location(id: 1) {
    id
  }
  episodesByIds(ids: [1, 2]) {
    id
  }
}

¿Nuevo en GraphQL? consulta la documentación aquí.

REST

URL base: https://rickandmortyapi.com/api

La URL base contiene información sobre todos los recursos disponibles de la API. Todas las peticiones son de tipo GET y van sobre https. Todas las respuestas devuelven datos en formato json.

JSON 
GET https://rickandmortyapi.com/api

{
  "characters": "https://rickandmortyapi.com/api/character",
  "locations": "https://rickandmortyapi.com/api/location",
  "episodes": "https://rickandmortyapi.com/api/episode"
}

Actualmente hay tres recursos disponibles:

  • Personaje: para obtener todos los personajes.
  • Ubicación: para obtener todas las ubicaciones.
  • Episodio: para obtener todos los episodios.

Cliente JavaScript

El cliente JavaScript de la Rick and Morty API es un cliente totalmente tipado que te da acceso a la API y todas sus funciones.

Ver en GitHub Referencia del cliente

Info y Paginación

La API paginará automáticamente las respuestas. Recibirás hasta 20 documentos por página.

Cada recurso contiene un objeto info con información sobre la respuesta.

Clave Tipo Descripción
countintLa cantidad total de resultados
pagesintLa cantidad de páginas disponibles
nextstring (url)Enlace a la siguiente página (si existe)
prevstring (url)Enlace a la página anterior (si existe)
JSON 
GET https://rickandmortyapi.com/api/character

{
  "info": {
    "count": 826,
    "pages": 42,
    "next": "https://rickandmortyapi.com/api/character?page=2",
    "prev": null
  },
  "results": [
    // ...
  ]
}

Personaje

Esquema de personaje

ClaveTipoDescripción
idintEl id del personaje
namestringEl nombre del personaje
statusstringEl estado del personaje ('Alive', 'Dead' o 'unknown')
speciesstringLa especie del personaje
typestringEl tipo o subespecie del personaje
genderstringEl género del personaje ('Female', 'Male', 'Genderless' o 'unknown')
originobjectNombre y enlace a la ubicación de origen del personaje
locationobjectNombre y enlace a la última ubicación conocida del personaje
imagestring (url)Enlace a la imagen del personaje (300x300px)
episodearray (urls)Lista de episodios en los que apareció el personaje
urlstring (url)Enlace al propio endpoint del personaje
createdstringFecha en que el personaje fue creado en la base de datos

Obtener varios personajes

Puedes obtener múltiples personajes añadiendo un array de ids como parámetro: /character/[1,2,3]

HTTP

Respuesta JSON

  {
    "id": 1,
    "name": "Rick Sanchez",
    // ...
  },
  {
    "id": 183,
    "name": "Johnny Depp",
    // ...
  }

Filtrar personajes

También puedes incluir filtros en la URL añadiendo parámetros de consulta adicionales. Para comenzar a filtrar, añade un ? seguido de la consulta <parámetro>=<valor>. Si quieres encadenar varias consultas en la misma llamada, usa & seguido del siguiente parámetro.

Parámetros disponibles:

ParámetroTipoDescripción
namestringFiltrar por el nombre dado
statusstringFiltrar por el estado dado (alive, dead o unknown)
speciesstringFiltrar por la especie dada
typestringFiltrar por el tipo dado
genderstringFiltrar por el género dado (female, male, genderless o unknown)
HTTP
GET https://rickandmortyapi.com/api/character/?name=rick&status=alive
